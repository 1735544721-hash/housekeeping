第 2 章 相关理论与技术

2.1 Spring Boot 框架介绍

Spring Boot 是 Spring 技术栈的轻量化封装，强调“约定优于配置”，通过自动配置与 starter 依赖快速完成 Web 与数据访问能力的组合，降低传统 XML 配置的复杂度。本系统后端采用 Spring Boot 3.2 与 Java 17 构建，借助 spring-boot-starter-web 提供 REST 接口，结合 Spring Security 与 JWT 形成统一登录鉴权链路，并以 MyBatis-Plus 完成 ORM 映射与分页查询；支付模块引入 Alipay SDK，消息通知通过 WebSocket 推送，接口文档由 Knife4j 输出，常用工具由 Hutool 提供。系统采用 Controller、Service、Mapper 三层架构划分职责，配合统一异常处理与 Actuator 监控机制，使服务项目、服务人员、订单、退款与评价等业务流程具备清晰边界与可维护性。

2.2 Vue.js 前端框架

Vue.js 是渐进式 JavaScript 前端框架，具有组件化开发、响应式数据驱动与单文件组件特性，适合构建高复用的单页面应用。

本系统前端采用 Vue 3 与 Vue CLI 5 搭建，使用 Vue Router 组织服务列表、服务详情、人员档案、下单支付、退款申诉、评价提交与个人中心等页面路由，结合 Pinia/Vuex 管理登录态与筛选条件，通过 Axios 统一封装接口请求；Element Plus 与图标库提供表格、表单、弹窗等交互组件，ECharts 用于统计与运营可视化，搭配 animate.css 与 dompurify/marked 提升页面表现与内容安全，使前端在高频浏览、预约与订单管理场景下保持稳定一致的交互体验。

2.3 MyBatis-Plus 数据访问框架

MyBatis-Plus 是对 MyBatis 的增强框架，保留原生 SQL 灵活性的同时提供通用 CRUD、条件构造器与分页能力，能够减少大量重复映射代码并提升数据层开发效率。其轻量化设计适合与 Spring Boot 集成，在保持查询可控的前提下，支持代码生成与统一规范，满足家政服务业务对数据一致性与可扩展性的要求。

系统通过 mybatis-plus-spring-boot3-starter 集成数据访问层，Mapper 统一继承 BaseMapper 以提供标准增删改查能力，服务层结合 LambdaQueryWrapper 与 Page 完成服务项目、人员排班、订单列表与评价记录的多条件分页查询。实体类使用 @TableName、@TableId 与 @TableField 建立字段映射，配合下划线转驼峰策略保证模型一致；针对统计报表、退款追踪与订单闭环等复杂场景，系统在 XML 中补充自定义 SQL，以兼顾查询性能与业务表达。

2.4 MySQL 数据库

MySQL 作为成熟的关系型数据库管理系统，具备完善的事务支持、索引机制与行级锁控制能力，适合承载家政服务订单与人员数据的高并发读写。本系统采用 InnoDB 引擎与 utf8mb4 编码建立 housekeeping_db 数据库，通过自增主键、唯一索引与业务索引保障用户账号、服务项目编号与订单号的唯一性与查询效率，并利用事务机制确保下单、支付、退款与评价等关键数据在状态切换时保持一致与可追溯。针对运营统计与历史查询场景，系统设置组合索引与时间字段优化，确保高频列表查询与报表汇总的稳定性能。

2.5 Java 语言

Java 是跨平台的面向对象语言，具备成熟的类库与完善的生态工具链，适合构建结构清晰、可维护性强的企业级应用。系统后端基于 Java 17 开发，利用类与接口组织业务模型，配合异常处理与泛型集合提升代码的稳健性与复用性；时间处理使用 LocalDate 与 LocalDateTime 保证预约时间与服务履约记录的语义准确。项目中广泛使用 Stream API 与 Optional 简化集合过滤、排序与聚合逻辑，并结合 Bean Validation 与自定义校验保障输入合法性；在订单支付、退款回滚与评价更新等关键流程中使用事务注解控制一致性，使家政服务系统在持续迭代中保持清晰的语言基础与工程可维护性。
