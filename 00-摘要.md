家政服务场景中需求发布、服务匹配、下单支付与评价环节涉及用户、服务人员与平台多方协同，现有线下或分散式管理方式容易出现信息滞后、服务标准不一、订单流转不清、投诉与退款追踪困难等问题，导致供需匹配效率低、服务过程难以形成闭环，运营统计口径也缺乏一致性。本文面向家政服务流程，设计并实现前后端分离的家政服务管理系统；前端采用 Vue 3、Element Plus 与 ECharts 构建服务展示、推荐与搜索、预约下单、评价提交、浏览收藏与个人中心等交互界面；后端基于 Spring Boot 3.2、Spring Security、MyBatis-Plus、JWT 与 MySQL 搭建统一业务层；系统覆盖用户与角色、菜单权限、服务分类、服务项目、服务人员、订单状态与退款、支付对接、评价、轮播、浏览历史、收藏、统计报表、文件上传与数据迁移等模块；并通过权限控制、交易状态与审计数据的统一治理提升订单闭环与运营统计能力，为家政服务供需对接与规范化运营提供支撑，支持平台日常运营与服务质量监管，适用于真实服务场景。关键词：家政服务系统；服务订单；服务人员；Spring Boot
